# Linux & Shell

## Summary
Linux is the foundation of most Telco cloud infrastructure. Mastering Linux administration, shell scripting, systemd, and troubleshooting techniques is essential for any DevOps engineer working in the Telco domain.

## Key Concepts

### System Administration
- **User & Permission Management**: Understanding user accounts, groups, sudo, and file permissions
- **Process Management**: ps, top, htop, kill, nice, renice
- **Package Management**: apt, yum, dnf - installing and managing software packages
- **File System**: ext4, xfs, mounting, fstab, disk management

### systemd
- **Service Management**: systemctl start/stop/restart/status
- **Unit Files**: Creating and managing service units
- **Targets**: Multi-user, graphical, rescue targets
- **Dependencies**: Requires, Wants, Before, After

### journald & Logging
- **journalctl**: Viewing and filtering system logs
- **Log Rotation**: Managing log sizes and retention
- **Syslog Integration**: Forwarding logs to centralized systems

### Shell Scripting
- **Bash Basics**: Variables, loops, conditionals, functions
- **Text Processing**: grep, sed, awk, cut, sort
- **Automation**: Cron jobs, at commands
- **Best Practices**: Error handling, input validation, logging

## Essential Tasks

1. **Navigate and manage the file system efficiently**
2. **Monitor system resources** (CPU, memory, disk, network)
3. **Troubleshoot system issues** using logs and diagnostics
4. **Write automation scripts** for repetitive tasks
